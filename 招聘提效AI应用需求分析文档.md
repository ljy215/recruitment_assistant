# 基于 AI 的招聘数据自动化记录与可视化系统需求分析文档

## 1. 项目背景

在现有招聘流程中，HR 需要在企业微信、企业微信群、腾讯在线文档等多个工作载体之间频繁切换，手动记录候选人信息、跟进面试进度、同步面试反馈、统计招聘数据。该流程存在以下典型问题：

- 人工录入效率低：简历信息、面试安排、面试反馈、候选人状态等数据需要重复复制、整理和更新。
- 数据容易遗漏：群聊消息、在线文档、会议纪要、面试官反馈分散在不同渠道，容易出现漏记、错记、延迟同步。
- 时效性差：招聘状态依赖 HR 人工维护，业务部门无法实时查看候选人进展。
- 统计复盘困难：通过率、淘汰原因、岗位漏斗、候选人标签等数据需要人工汇总，影响招聘决策效率。

因此，本项目拟在最小成本预算下，设计并落地一套基于 AI 技术的招聘流程提效应用，实现招聘数据自动化记录、实时同步更新和可视化展示，替代 HR 的高频手工录入工作，提升招聘流程的自动化与智能化水平。

## 2. 项目目标

本项目目标是构建一个轻量级、低成本、可运行的 AI 招聘提效 Demo，围绕招聘全流程中的数据记录与跟踪痛点，完成以下能力：

1. 自动采集招聘过程中的关键数据，包括简历信息、群聊反馈、面试纪要、人工评分等。
2. 利用 AI 模型完成简历解析、关键信息抽取、候选人标签生成、面试反馈总结和淘汰原因归类。
3. 将结构化招聘数据自动同步至腾讯在线文档或数据库，减少 HR 手工录入。
4. 构建招聘数据看板，实时展示候选人状态、岗位进度、面试结果、通过率和主要淘汰原因。
5. 在保证成本可控的前提下，优先使用企业已有工具和轻量化 AI 能力，降低系统建设和维护成本。

## 3. 用户与角色分析

### 3.1 HR

HR 是系统的核心使用者，主要关注招聘流程是否能够被自动记录、自动跟踪和及时提醒。

核心需求：

- 自动解析简历，减少基础信息录入。
- 自动同步面试安排、面试反馈和候选人状态。
- 快速查看候选人当前流程节点。
- 自动生成招聘统计报表和复盘材料。

### 3.2 面试官

面试官主要参与面试评价和反馈提交。

核心需求：

- 面试后快速提交评分和评价。
- 能够查看 AI 提取的面试纪要和候选人关键信息。
- 减少重复填写长篇反馈的负担。

### 3.3 用人部门负责人

用人部门负责人关注招聘进度和候选人与岗位的匹配程度。

核心需求：

- 实时查看岗位招聘漏斗。
- 快速了解候选人优势、风险和淘汰原因。
- 基于数据辅助做出推进或淘汰决策。

### 3.4 系统管理员

系统管理员负责配置岗位、技能标签、数据权限和 AI 调用参数。

核心需求：

- 配置岗位要求和候选人评价维度。
- 管理 AI 技能库和提示词模板。
- 控制数据访问权限，保障候选人隐私安全。

## 4. 核心痛点分析

| 痛点 | 当前表现 | 影响 |
| --- | --- | --- |
| 手工录入重复 | HR 需从简历、微信群、会议纪要中复制信息到在线文档 | 耗时长，易出错 |
| 状态更新滞后 | 候选人是否通过、是否约面、是否发 offer 依赖人工维护 | 业务部门无法实时掌握进度 |
| 信息分散 | 数据分布在企业微信、群聊、腾讯文档、会议纪要中 | 形成信息孤岛 |
| 反馈不标准 | 面试官反馈格式不统一，原因描述主观且难以统计 | 后续复盘困难 |
| 数据可视化弱 | 通过率、淘汰原因、岗位进度需要人工统计 | 决策效率低 |
| 成本与隐私压力 | 完全依赖外部大模型可能带来成本和数据合规风险 | 难以长期推广 |

## 5. 业务流程分析

### 5.1 面试前流程

现有流程：

1. HR 收到候选人简历。
2. HR 手动打开简历并提取姓名、电话、邮箱、学校、工作年限等信息。
3. HR 将信息复制到腾讯在线文档。
4. HR 根据岗位要求初步判断是否匹配。
5. HR 在企业微信群或私聊中同步候选人情况。

优化后流程：

1. HR 上传 PDF/Word 简历至系统。
2. AI 自动解析简历文本。
3. 系统抽取候选人基础信息和核心经历。
4. AI 根据岗位要求生成匹配度评分和标签。
5. 系统自动写入腾讯在线文档或后台数据库。
6. 系统生成候选人摘要和推荐动作，HR 可复制到企业微信群或腾讯在线文档中同步。

### 5.2 面试中流程

现有流程：

1. 面试通过腾讯会议进行。
2. 面试官或 HR 手动记录关键问题和候选人回答。
3. 面试结束后，面试官在文档或群里填写反馈。
4. HR 手动整理反馈并更新候选人状态。

优化后流程：

1. 腾讯会议智能纪要生成面试文本。
2. 系统导入或接收会议纪要。
3. AI 提取关键问答、能力表现、风险点和面试结论。
4. 面试官在系统中进行人工评分和确认。
5. 系统结合 AI 摘要与人工评分生成结构化面试结果。
6. 候选人状态自动更新为“通过”“待定”或“淘汰”。

### 5.3 面试后流程

现有流程：

1. HR 手动汇总面试结果。
2. HR 更新腾讯在线文档中的候选人状态。
3. HR 统计通过率、淘汰原因、岗位进度等数据。
4. 用人部门通过群聊或文档了解进展。

优化后流程：

1. 系统自动生成候选人 Markdown 评估报告。
2. 系统将结构化结果同步至腾讯在线文档。
3. 数据看板实时展示招聘漏斗、候选人标签和淘汰原因。
4. 系统生成可复制的状态更新文案，HR 可按需同步到企业微信群。
5. HR 只需审核异常信息和关键决策节点。

## 6. 功能需求

### 6.1 简历上传与解析模块

功能描述：

支持 HR 上传候选人简历，系统自动提取文本并调用 AI 模型进行结构化解析。

输入：

- PDF 简历
- Word 简历
- 岗位名称
- 岗位要求

输出：

- 姓名
- 手机号
- 邮箱
- 年龄或毕业年份
- 学历
- 学校
- 专业
- 工作年限
- 最近工作单位
- 核心项目经历
- 技能关键词
- 岗位匹配度
- 初筛建议

关键要求：

- 支持常见 PDF 简历文本提取。
- 字段缺失时标记为“待补充”，避免 AI 编造。
- 输出结构化 JSON，便于写入数据库和在线文档。
- 支持人工修改和确认。

### 6.2 RAG 岗位知识库模块

功能描述：

将岗位 JD、业务要求、面试评价标准、历史优秀候选人特征等资料构建为轻量知识库，供 AI 在简历分析和面试总结时检索使用。

知识库内容：

- 岗位说明书
- 技能要求
- 业务场景说明
- 面试题库
- 评价维度
- 历史通过/淘汰原因样例

关键要求：

- 支持按岗位检索相关要求。
- 支持对候选人简历进行岗位匹配分析。
- 支持低成本向量数据库，如 Chroma、FAISS 或本地 SQLite + embedding 缓存。
- 初期 Demo 可使用本地 Markdown/JSON 文件模拟知识库。

### 6.3 AI 技能库模块

功能描述：

将不同岗位或评价维度封装为可复用的 AI 技能，便于按场景调用。

建议技能：

| 技能名称 | 适用场景 | 输出结果 |
| --- | --- | --- |
| 简历基础信息抽取技能 | 简历上传后 | 候选人结构化信息 |
| 技术栈匹配技能 | 技术岗初筛 | 技术关键词、技术深度、匹配度 |
| 沟通表达评估技能 | 销售、运营、HRBP 等岗位 | 沟通能力标签、表达风险点 |
| 稳定性分析技能 | 所有岗位 | 跳槽频率、职业连续性风险 |
| 面试纪要总结技能 | 面试后 | 关键问答、能力表现、风险点 |
| 淘汰原因归类技能 | 面试结果归档 | 标准化淘汰原因 |
| 候选人报告生成技能 | 面试结束后 | Markdown 评估报告 |

关键要求：

- 技能以提示词模板 + 输入参数 + 输出 JSON Schema 的形式管理。
- 支持按岗位自动选择技能。
- 支持人工调整提示词，方便小组 Demo 展示。

### 6.4 企业微信/群消息同步模块

功能描述：

由于当前没有企业微信机器人权限，Demo 阶段暂不接入企业微信机器人或 Webhook。系统先生成标准化的群消息文案，由 HR 一键复制后发送到企业微信群。后续如果企业开通机器人权限，再扩展为自动推送。

典型同步场景：

- 新候选人已录入。
- 候选人初筛通过，待安排面试。
- 面试纪要已生成，待面试官确认。
- 候选人状态变更。
- 招聘数据日报/周报生成。

关键要求：

- Demo 阶段不依赖企业微信机器人权限。
- 系统提供“一键复制群消息”能力。
- 同步内容应简洁，包括候选人姓名、岗位、状态、关键标签和下一步动作。
- 避免在群消息中展示敏感隐私信息，如完整手机号、身份证号等。
- 企业微信机器人 Webhook 作为后续扩展功能，不纳入当前 MVP 必做范围。

### 6.5 腾讯在线文档同步模块

功能描述：

将系统生成的结构化招聘数据同步到腾讯在线文档，保留 HR 现有工作习惯。

同步字段：

- 候选人姓名
- 应聘岗位
- 联系方式脱敏展示
- 简历来源
- 当前状态
- 初筛结果
- 面试时间
- 面试官
- 面试评分
- AI 总结
- 淘汰/通过原因
- 下一步动作

关键要求：

- 若无法直接调用腾讯文档开放接口，Demo 可先导出为 CSV/Excel。
- 保持字段结构稳定，便于 HR 继续使用在线文档。
- 支持增量更新，避免重复写入候选人。

### 6.6 面试纪要分析模块

功能描述：

基于腾讯会议智能纪要或手动上传的面试文本，AI 自动提取面试过程中的关键信息。

输入：

- 会议纪要文本
- 候选人简历摘要
- 岗位要求
- 面试官人工评分

输出：

- 面试关键问答
- 技术/业务能力评价
- 沟通表达评价
- 风险点
- 候选人与岗位匹配结论
- 建议状态
- 标准化通过/淘汰原因

关键要求：

- AI 结论必须与原始纪要可追溯。
- 保留人工确认机制，避免完全依赖 AI 决策。
- 对面试官评分和 AI 结论不一致的情况进行标记。

### 6.7 人在循环审核模块

功能描述：

系统不直接替代招聘决策，而是辅助 HR 和面试官完成数据整理、信息提取和决策支持。关键节点由人工确认。

需要人工确认的节点：

- 简历解析结果确认
- 初筛是否通过
- 面试评分
- 最终通过/淘汰状态
- offer 推进动作

关键要求：

- AI 输出默认标记为“待确认”。
- 人工确认后才能进入正式数据看板。
- 保留修改记录，便于复盘。

### 6.8 数据看板模块

功能描述：

构建招聘数据可视化看板，帮助 HR 和业务部门实时查看招聘状态。

核心指标：

- 候选人总数
- 各岗位候选人数量
- 初筛通过率
- 面试通过率
- offer 转化率
- 当前待处理候选人数
- 平均处理时长
- 高频淘汰原因
- 候选人标签分布

展示方式：

- 招聘漏斗图
- 岗位进度表
- 候选人状态看板
- 标签分布图
- 淘汰原因柱状图
- 每日/每周招聘数据摘要

关键要求：

- Demo 阶段采用 React + ECharts 或 Vue3 + ECharts 实现。
- 数据来源可为 SQLite、CSV 或腾讯在线文档导出数据。
- 支持按岗位、状态、时间范围筛选。

### 6.9 Markdown 岗位候选人汇总报告模块

功能描述：

系统针对某一岗位的最终筛选名单，自动生成简洁的 Markdown 汇总报告。例如某岗位有 10 名候选人进入最终筛选，系统需在一份报告中集中展示这 10 名候选人的基础信息、面试表现、AI 标签、综合评分和推荐结论，方便 HR 与用人部门快速横向比较。

报告结构：

```markdown
# XX岗位最终候选人筛选报告

| 排名 | 姓名 | 核心背景 | 面试表现摘要 | 优势标签 | 风险点 | 综合评分 | 建议 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 张三 | 3年售前经验，本科 | 表达清晰，能结合项目说明客户需求分析过程 | 沟通强、方案能力好 | 技术深度待确认 | 88 | 优先推进 |
| 2 | 李四 | 5年项目管理经验，硕士 | 逻辑完整，过往项目复杂度较高 | 项目经验强、稳定性好 | 行业经验略弱 | 84 | 可进入终面 |
```

输出字段：

- 候选人基础信息：姓名、岗位、学历、工作年限、最近经历。
- 面试表现摘要：用 1 句话概括候选人在面试中的核心表现。
- 优势标签：最多 3 个标签，如“技术扎实”“沟通强”“稳定性好”。
- 风险点：最多 1 到 2 个风险点，缺失则写“暂无明显风险”。
- 综合评分：结合简历匹配度、AI 分析和面试官评分，输出 0-100 分。
- 推荐结论：优先推进、可进入终面、待定、不建议推进。

关键要求：

- 报告必须简洁，重点支持横向对比，不展开长篇个人分析。
- 每位候选人的面试表现控制在 1 句话内。
- 支持按综合评分自动排序。
- AI 评分仅作为辅助参考，最终结论需保留人工确认。
- 支持一键导出 Markdown，便于复制到腾讯在线文档或群内同步。

## 7. 非功能需求

### 7.1 成本控制需求

本方案需体现最小成本预算原则：

- 优先复用企业已有工具：企业微信、腾讯会议、腾讯在线文档。
- Demo 阶段优先使用开源框架：React/Vue、FastAPI、SQLite、Chroma、FAISS。
- AI 模型采用“外部大模型 + 本地轻量模型”的组合策略。
- 对高频、低复杂度任务使用本地模型或规则模板处理。
- 对复杂总结和语义判断任务调用大模型。
- 对简历 embedding 和岗位知识库结果进行缓存，减少重复调用成本。

### 7.2 数据安全需求

- 候选人手机号、邮箱等敏感信息需脱敏展示。
- 简历原文、会议纪要等文件需设置访问权限。
- AI 调用前应尽量去除不必要的隐私字段。
- 支持本地化部署，避免核心招聘数据长期暴露在外部平台。
- 系统应保留操作日志，记录谁在何时查看或修改了候选人信息。

### 7.3 可用性需求

- HR 操作流程应尽量简单，上传简历后即可自动生成结果。
- 数据看板应清晰展示“谁需要处理什么”。
- AI 生成内容允许人工修改。
- 系统异常时不能影响原有腾讯文档流程。

### 7.4 可扩展性需求

- 支持新增岗位。
- 支持新增 AI 技能。
- 支持新增评价维度。
- 支持后续对接 ATS、OA 或企业内部系统。

## 8. 系统架构设计

### 8.1 总体架构

系统采用轻量化架构，由前端页面、后端服务、AI 能力层、数据存储层和外部工具接入层组成。

```text
HR/面试官
   |
前端 Demo 页面 React/Vue
   |
后端服务 FastAPI
   |
--------------------------------
| 简历解析 | AI 技能调度 | 面试纪要分析 |
--------------------------------
   |
AI 模型层：大模型 API / 本地小模型 / Embedding 模型
   |
数据层：SQLite / CSV / 向量库 / 腾讯在线文档
   |
外部工具：腾讯会议纪要 / 腾讯在线文档 / 企业微信群人工同步
```

### 8.2 推荐技术选型

| 模块 | 推荐技术 | 原因 |
| --- | --- | --- |
| 前端 Demo | React + Vite 或 Vue3 + Vite | 页面开发灵活，适合实现候选人管理、看板和报告预览 |
| 后端服务 | FastAPI | 轻量、接口清晰 |
| 数据库 | SQLite | 零部署、低成本 |
| 简历解析 | PyMuPDF / pdfplumber / python-docx | 开源且易集成 |
| AI 调用 | OpenAI API / 通义千问 / DeepSeek / 本地模型 | 可按成本和隐私选择 |
| RAG | Chroma / FAISS | 本地向量检索成本低 |
| 可视化 | ECharts | 适合在 Vue/React 中实现招聘漏斗、标签分布和淘汰原因统计 |
| 消息同步 | 一键复制群消息文案 | 不依赖企业微信机器人权限，适合 Demo 落地 |
| 文档同步 | CSV/Excel 导出或腾讯文档 API | 保留现有工作习惯 |

## 9. AI 应用设计

### 9.1 AI Agent 工作流

系统中的 AI Agent 负责根据招聘流程自动选择任务和工具。

典型流程：

1. 接收简历上传事件。
2. 调用简历文本提取工具。
3. 调用岗位知识库检索工具。
4. 调用简历结构化抽取技能。
5. 调用岗位匹配评估技能。
6. 写入数据库和在线文档。
7. 生成企业微信群同步文案，供 HR 复制发送。
8. 接收面试纪要。
9. 调用面试总结技能和淘汰原因归类技能。
10. 生成候选人报告和数据看板更新。

### 9.2 Prompt 模板示例

```text
你是招聘数据分析助手。请基于候选人简历文本和岗位要求，提取结构化信息。

要求：
1. 只提取原文中明确存在的信息，不要编造。
2. 缺失字段填写“待补充”。
3. 输出 JSON 格式。
4. 给出岗位匹配度评分，范围 0-100。
5. 给出最多 5 个候选人标签。

输入：
岗位要求：{job_description}
简历文本：{resume_text}
```

## 15. 风险分析与应对措施

| 风险 | 表现 | 应对措施 |
| --- | --- | --- |
| AI 抽取不准确 | 简历格式复杂导致字段错误 | 增加人工确认和字段置信度 |
| AI 编造信息 | 原文无信息但模型补全 | Prompt 明确禁止编造，缺失填“待补充” |
| 外部 API 成本过高 | 高频调用大模型 | 缓存结果，本地模型处理简单任务 |
| 数据隐私风险 | 简历和纪要包含敏感信息 | 脱敏后调用模型，支持本地部署 |
| 腾讯文档接口限制 | API 权限获取困难 | Demo 阶段先使用 CSV/Excel 导出 |
| 企业微信机器人权限不足 | 无法获取 Webhook，不能自动推送 | Demo 阶段改为生成群消息文案，由 HR 一键复制同步 |
| 面试官不愿使用新系统 | 操作负担增加 | 保留企业微信提醒和文档习惯，减少额外录入 |

## 18. 总结

本方案围绕 HR 在招聘流程中“大量手动记录与跟踪招聘数据”的核心痛点，基于企业微信、企业微信群、腾讯会议和腾讯在线文档等现有工作载体，设计了一套轻量级 AI 招聘提效系统。

系统通过简历解析、RAG 岗位知识库、AI 技能库、面试纪要分析、人工确认、数据同步和可视化看板，形成从面试前、面试中到面试后的完整闭环。在技术实现上，方案优先采用开源框架和低成本模型调用方式，并保留本地化部署可能性，在成本、效率和数据安全之间取得平衡。

该方案不仅能够替代 HR 的重复性手工录入工作，也能够帮助招聘团队沉淀结构化数据，提升招聘决策的科学性和流程管理效率。
